# Code Review Skill

## When to Use

Use this skill when reviewing code changes for quality, security, and maintainability.

## Review Checklist

### Functionality
- [ ] Code accomplishes its stated purpose
- [ ] Edge cases are handled appropriately
- [ ] Error handling is comprehensive

### Code Quality
- [ ] Code follows project conventions
- [ ] No unnecessary complexity
- [ ] Clear naming and structure

### Security
- [ ] No hardcoded secrets
- [ ] Input validation present
- [ ] No injection vulnerabilities

### Testing
- [ ] Adequate test coverage
- [ ] Tests are meaningful, not just present
- [ ] Edge cases tested

## Review Process

1. Understand the context and purpose
2. Read through changes systematically
3. Check against review checklist
4. Provide constructive feedback
5. Verify fixes if requested
